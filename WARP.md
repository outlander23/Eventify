# WARP.md

This file provides guidance to WARP (warp.dev) when working with code in this repository.

## Project Overview

This is EventHub, a Progressive Web App (PWA) built with React, Vite, and TypeScript for event registration and management. The application supports both online and offline functionality, with background sync capabilities for registration queuing when offline.

## Development Commands

### Essential Commands
- `npm run dev` - Start development server with hot reload
- `npm run build` - Build for production (includes PWA service worker generation)
- `npm run preview` - Preview production build locally
- `npm run lint` - Run ESLint (Note: ESLint not currently installed in dependencies)

### Testing Individual Components
Since there's no test framework configured, manual testing is done via:
- `npm run dev` and navigate to specific routes
- Use browser DevTools to test PWA features (offline mode, service worker)

## Architecture Overview

### Core Structure
- **Frontend Framework**: React 18 with React Router for client-side routing
- **Build Tool**: Vite with PWA plugin for service worker generation
- **Styling**: Tailwind CSS with custom design system using Radix UI components
- **State Management**: React Context (AuthProvider, PWAProvider)
- **Authentication**: Mock authentication with localStorage persistence
- **PWA Features**: Service worker, background sync, push notifications, install prompts

### Key Directories
- `src/pages/` - Route components including admin dashboard pages
- `src/components/ui/` - Reusable Radix UI-based components
- `src/components/pwa/` - PWA-specific components (install prompts, offline status)
- `src/components/events/` - Event-related components
- `src/components/layout/` - Header, footer, and layout components
- `src/lib/` - Utility libraries (auth, PWA utilities, analytics, notifications)
- `src/hooks/` - Custom React hooks

### Authentication System
The app uses a mock authentication system that stores user data in localStorage. Users can sign in with any email, and admin access is granted for emails containing "admin". The system supports:
- Sign in/up with email
- Role-based access (admin/user)
- Profile management
- Session persistence

### PWA Implementation
The app implements full PWA capabilities:
- **Service Worker**: Generated by Vite PWA plugin with caching strategies
- **Offline Support**: Queued registrations stored in IndexedDB via service worker
- **Background Sync**: Syncs queued registrations when back online
- **Install Prompts**: Custom install promotion UI
- **Push Notifications**: VAPID-based push notification system
- **Offline Indicator**: Visual feedback for connectivity status

### Routing Structure
- `/` - Homepage
- `/events` - Event listing
- `/events/:slug` - Event details
- `/events/:slug/register` - Registration form
- `/auth` - Authentication
- `/account/*` - User account management
- `/admin/*` - Admin dashboard and management tools
- `/offline` - Offline fallback page

## Development Patterns

### Component Architecture
- Uses shadcn/ui component library built on Radix UI
- Consistent use of Tailwind CSS utility classes
- Components follow compound component pattern for complex UI elements
- Form handling with React Hook Form and Zod validation

### State Management
- React Context for global state (auth, PWA status)
- Local component state for UI interactions
- localStorage for client-side persistence
- Service worker for offline data queuing

### PWA-Specific Development
- PWA utilities in `src/lib/pwa-utils.ts` for service worker interaction
- Background sync registration for offline functionality
- Network status monitoring
- Installation state detection

## File Conventions
- React components use `.jsx` extension (main application files)
- TypeScript files use `.ts`/`.tsx` extensions (utilities and newer components)
- Path aliases: `@/*` maps to `./src/*`
- Component names use PascalCase
- Utility functions use camelCase

## Build Configuration
- **Vite Config**: Custom PWA manifest, service worker settings, and path resolution
- **TypeScript**: Strict mode enabled with path mapping
- **Tailwind**: Custom color palette and component classes
- **PWA Manifest**: Configured for "EventHub" with standalone display mode
